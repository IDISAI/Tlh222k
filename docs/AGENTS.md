# docs - agent notes

Documentation edits should make current behavior clear and label roadmap/future
behavior explicitly.

- Do not present unbuilt systems as current production code.
- Keep env guidance in `docs/onboarding/env.md` aligned with every committed
  `.env.example`.
- If code changes alter commands, architecture, env, deploy, or package
  boundaries, update docs in the same change.
- Prefer ASCII in new docs unless the surrounding file intentionally uses
  Vietnamese text with accents.
